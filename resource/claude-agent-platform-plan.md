# Claude Agent 기반 Agent 관리 플랫폼 기획서

> Version 0.2 | 고도화 반영 | 2026-05-13

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적

사용자가 요구사항을 자연어 대화로 정리하고, 기술 스택/언어/도메인/프로젝트 방향을 선택한 뒤 최종 요약을 검토·승인하면, Claude Sub-Agent 팀이 자동으로 POC 코드를 생성하는 플랫폼을 구축한다.

핵심 철학: **대화형 요구사항 수집 + 승인 기반 실행 게이트 + 멀티 에이전트 자동 오케스트레이션**.

### 1.2 주요 지원 시나리오

| 요구 유형 | 설명 | 예시 |
|---|---|---|
| 신규 구축 | 새로운 서비스/시스템 개발 | "FastAPI + React 쇼핑몰 구축" |
| 고도화 | 기존 코드베이스 기능 추가 | "기존 API에 JWT 인증 추가" |
| 리팩토링 | 코드 품질 개선 | "레거시 Java 서비스 Spring Boot 전환" |
| POC | 개념 증명 빠른 프로토타입 | "AI 챗봇 MVP 2일 안에" |

## 2. 요구사항 수집/승인 중심 신규 플로우

### 2.1 단계별 사용자 플로우 (신규)

1. **대화형 요구사항 수집**
- 웹 UI에서 사용자와 대화
- 대화 모델 선택
  - 내부 LLM (플랫폼 내장)
  - 외부 LLM (환경변수 기반 연동: 예 `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `AZURE_OPENAI_*`)
- 대화 결과를 구조화 요구사항으로 변환

2. **프로젝트 옵션 선택**
- 기술 스택 선택: `web`, `backend`, `db`
- 사용 언어 선택: `C`, `C++`, `Java`, `Python` (복수 선택 가능)
- 프로젝트 방향 선택: `신규`, `고도화`, `리팩토링`, `POC`
- 비즈니스 영역 선택: `금융`, `유통`, `공공`, `B2B`, `B2C`

3. **최종 요약본 생성 및 사용자 검토**
- 시스템이 대화 결과 + 옵션을 합쳐 실행 요약본 생성
- 사용자에게 명시적 검토 요청

4. **승인 게이트**
- 사용자 `OK` 전까지 코드 생성 시작 금지
- `수정 요청` 시 요약 단계로 복귀

5. **POC 코드 생성 시작**
- 승인 후 오케스트레이터 실행
- Sub-Agent 팀 자동 편성 및 코드 생성

### 2.2 승인 정책

- 실행 상태는 `draft -> ready_for_review -> approved -> running -> packaged` 순으로 전이
- `approved` 상태 전환 이벤트를 POC 생성 트리거로 사용
- 모든 승인/반려 이력은 감사 로그로 저장

## 3. 시스템 아키텍처 (변경점)

### 3.1 LLM 대화 레이어 추가

```
[사용자 웹 UI]
      |
      | Chat / Form
      v
[Requirement Conversation Service]
      |
      |-- Internal LLM Adapter
      |-- External LLM Adapter (Env-configured)
      |
      v
[Requirement Normalizer]
      |
      v
[Review & Approval Gate]
      |
      v
[Orchestrator + Agent SDK Runtime]
```

### 3.2 외부 LLM 연동 구성 원칙

- 외부 LLM Provider 설정은 서버 환경변수로만 관리
- 클라이언트에 API 키 전달 금지
- Provider별 Adapter 패턴으로 구현
  - `internal`
  - `openai`
  - `anthropic`
  - `azure-openai`

## 4. Agent 팀 구성 로직

### 4.1 요구사항 분석 → 팀 자동 편성

- 승인 이전: `spec-analyst` 중심의 요구사항 정제만 수행
- 승인 이후: `orchestrator`가 방향/스택/언어/도메인을 반영해 팀 구성

예시:
- 신규 + 금융 + web/backend/db + Python/Java
  - `spec-analyst -> backend-architect + frontend-developer + database-architect (parallel) -> security-auditor -> test-automator`
- 고도화 + 공공
  - `spec-analyst -> code-reviewer -> 관련 도메인 에이전트 -> test-automator`
