# To-Do 앱 서비스 기획서

## 1. 서비스 개요

할 일 목록을 관리하고 마감일을 설정하여 작업을 효율적으로 추적할 수 있는 웹 기반 To-Do 애플리케이션

### 핵심 기능
- 할 일 생성, 조회, 수정, 삭제
- 마감일 설정 및 관리
- 완료 상태 토글
- 마감일 기준 정렬 및 필터링

---

## 2. 사용자 시나리오

### 시나리오 1: 할 일 추가
1. 사용자가 메인 화면에 접속
2. 할 일 제목 입력
3. 마감일 선택 (선택사항)
4. "추가" 버튼 클릭
5. 할 일이 목록에 표시됨

### 시나리오 2: 할 일 완료 처리
1. 할 일 목록에서 완료할 항목 확인
2. 체크박스 클릭
3. 완료 상태로 변경됨 (취소선 표시)

### 시나리오 3: 할 일 수정
1. 할 일 항목의 수정 버튼 클릭
2. 제목 또는 마감일 변경
3. "저장" 버튼 클릭
4. 변경사항이 반영됨

### 시나리오 4: 할 일 삭제
1. 삭제할 할 일 항목의 삭제 버튼 클릭
2. 확인 메시지 표시
3. 확인 시 목록에서 제거됨

## 4. 화면 상세

### 메인 화면

#### 입력 요소
- 할 일 제목 입력 필드 (텍스트 입력)
- 마감일 선택 필드 (날짜 선택기)

#### 버튼
- "추가" 버튼: 새 할 일 등록
- "수정" 버튼: 각 할 일 항목별로 수정 모드 전환
- "저장" 버튼: 수정 내용 저장
- "취소" 버튼: 수정 취소
- "삭제" 버튼: 할 일 삭제

#### 표시 요소
- 할 일 목록 (테이블 또는 카드 형식)
  - 체크박스 (완료/미완료 상태)
  - 할 일 제목
  - 마감일 (있는 경우)
  - 생성일
  - 수정/삭제 버튼

#### 동작
- 체크박스 클릭 시 완료 상태 토글
- 완료된 항목은 취소선 표시
- 마감일이 지난 항목은 빨간색으로 강조
- 수정 버튼 클릭 시 해당 항목이 편집 모드로 전환
- 삭제 버튼 클릭 시 확인 후 항목 삭제

---

## 5. 기능 목록

### F1. 할 일 생성
- 제목 필수 입력
- 마감일 선택 옵션
- 생성일 자동 기록

### F2. 할 일 목록 조회
- 전체 할 일 목록 표시
- 생성일 기준 최신순 정렬
- 완료/미완료 상태 표시

### F3. 할 일 수정
- 제목 수정
- 마감일 수정

### F4. 할 일 삭제
- 선택한 할 일 영구 삭제

### F5. 완료 상태 토글
- 완료/미완료 상태 변경
- 완료 시 취소선 표시

### F6. 마감일 관리
- 마감일 설정/해제
- 마감일 지난 항목 시각적 강조

## 6. API 요구사항

### GET /api/todos
- 설명: 모든 할 일 목록 조회
- 응답:
```json
[
  {
    "id": 1,
    "title": "프로젝트 완료하기",
    "completed": false,
    "deadline": "2026-05-15",
    "created_at": "2026-05-08T10:00:00Z"
  }
]
```

### POST /api/todos
- 설명: 새 할 일 생성
- 요청:
```json
{
  "title": "프로젝트 완료하기",
  "deadline": "2026-05-15"
}
```
- 응답:
```json
{
  "id": 1,
  "title": "프로젝트 완료하기",
  "completed": false,
  "deadline": "2026-05-15",
  "created_at": "2026-05-08T10:00:00Z"
}
```

### PUT /api/todos/:id
- 설명: 할 일 수정
- 요청:
```json
{
  "title": "프로젝트 수정하기",
  "deadline": "2026-05-20",
  "completed": true
}
```
- 응답:
```json
{
  "id": 1,
  "title": "프로젝트 수정하기",
  "completed": true,
  "deadline": "2026-05-20",
  "created_at": "2026-05-08T10:00:00Z"
}
```

### DELETE /api/todos/:id
- 설명: 할 일 삭제
- 응답:
```json
{
  "message": "삭제되었습니다"
}
```

### PATCH /api/todos/:id/toggle
- 설명: 완료 상태 토글
- 응답:
```json
{
  "id": 1,
  "completed": true
}
```

## 7. 데이터 요구사항

### 할 일 (Todo)
- ID (고유 식별자)
- 제목 (필수, 텍스트)
- 완료 여부 (기본값: false)
- 마감일 (선택, 날짜)
- 생성일 (자동 생성)

---

## 8. 비기능 요구사항

- 입력 검증: 제목은 빈 값 불가
- 에러 처리: 네트워크 오류 시 사용자에게 알림
- 로딩 상태: API 호출 중 로딩 표시
- 반응형: 모바일/데스크톱 모두 지원
